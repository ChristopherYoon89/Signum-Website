Documentation

The Signum News API provides structured access to aggregated news articles, sources, and categories. Results can be filtered, ordered and paginated using URL parameters.

Users of the API receive access to the Signum dashboard, where they can manage their account details and generate API keys. Keys are hashed before being stored in the database, and the full key is shown only once at the time of creation. Users are therefore responsible for storing their keys securely. If a key is lost, a new one can be generated at any time. Multiple API keys can be created to support different applications or environments.

Endpoint News Article

News article data can be retrieved from the following endpoint, which returns all articles, ordered by publication date by default. In practice, this provides a balanced mix of content across different categories.

GET /api/public/news-articles/

The following section provides a short description of the filters:

  • search_title: This field allows users to filter articles based on specific keywords and their occurrence in the article titles.

  • category_primary_name/category_secondary_name: Categories allow users to filter articles based on our content taxonomy. In general, categories are assigned by an artificial neural network as part of our data pipeline. In most cases, the algorithm identifies the two most likely categories, which are then set as the primary and secondary category.

  • tag: In addition to the primary and secondary categories, our data pipeline extracts key terms from both the title and the full text of each article. A separate tagging system then processes these keywords and converts them into standardized tags. The tag filter allows users to narrow down their search based on the top keywords of an article.

  • source_name: Filter articles by the name of the news source. You can use keywords to narrow results, or retrieve a detailed list of all available sources via the sources endpoint.

  • source_id: Field allows you to use a unique identifier to filter articles based on their source. The ID of the source could be used for dropdown menus or other application features.

  • date_from/date_to: The two date filters allows user to filter articles based on their publication date or upload date.

  • min_user_rating/max_user_rating: The user rating represents the average user rating between 1 (lowest rating) and 5 (highest rating). All articles with a rating of 0 were not rated by any user. The two fields allow you define a minimum or maximum user rating. The average user rating is a float field.

  • min_clicks_count/max_clicks_count: Another field, which allows users to filter articles based on their engagement level, is the number of recorded clicks. You can specify minimum and/or maximum click counts to narrow results to either highly viewed articles or less prominent content. This filter is an integer field.

  • min_algo_rating/max_algo_rating: To help distinguish signal from noise, our data pipeline includes an algorithmic rating system that ranks articles based on their quality and relevance. The ranking score ranges from 0 (lowest score) to 5 (highest score) and represents a float field.

  • min_source_user_rating/max_source_user_rating: These fields allow users to filter articles based on the overall quality of their source. The values reflect the average user rating across all articles published by a particular source, making it possible to include only highly rated publishers or exclude lower-rated ones.

  • min_source_article_count/max_source_article_count: These fields allow users to filter articles based on the publishing volume of their source. The values represent the total number of articles available from a particular source, making it possible to focus on high-volume publishers or limit results to more selective outlets.

  • min_source_algo_rating/max_source_algo_rating: These fields allow users to filter articles based on the overall algorithmic performance of their source. The values represent the average algorithmic rating across all articles published by a particular source, enabling users to focus on sources that consistently score higher according to the ranking system.

Single Article

In some cases, users may want to access individual articles. This can be done through the article endpoint by appending the article ID to the API URL. 

GET /api/public/news-articles/175/

Endpoint News Sources

The News Source model represents the publishers and platforms from which articles are collected. This makes it possible to identify high-performing sources, analyze coverage or build curated source lists.

We also advice our clients to use the News Source endpoint, if they want to filter articles based on the source name.

GET /api/public/news-sources/

News Source Fields

It provides aggregated metrics that describe each source, including user ratings, publishing volume and average algorithmic performance. The News Sources endpoint allows users to retrieve and explore available publishers, filter them by name or rating, and sort results based on quality or activity. 

{
  "id": 42,
  "name": "Wired",
  "viaplatform": "website",
  "average_rating": 3.3,
  "article_count": 43,
  "average_algo_rating": 2.7,
}

Filter Fields

These filtering fields allow users to narrow down news sources based on the name of the source, platform, quality and publishing activity. Sources can be filtered by name or platform, as well as by minimum and maximum average user rating. Additional filters make it possible to restrict results based on the average algorithmic rating of a source and the total number of articles it has published, enabling users to focus on high-quality or high-volume publishers.

min_article_count

  • source_name: This filter allows users to search articles by the name of the news source. You can use keywords for the search or the exact name of the source.

  • platform: The platform filter searches the viaplatform data field and allows users to distinguish between different publishing platforms. In most cases the viaplatform field of the News Source model includes the category Website, other examples are Twitter or Substack.

  • min_user_rating/max_user_rating: The filters search articles based on the user_rating field and allow users to filter sources based on the average user rating across all articles published by a particular source.

  • min_source_algo_rating/max_source_algo_rating: The min_source_algo_rating and max_source_algo_rating fields allow users to filter sources based on the average algorithmic rating of all articles published by a particular source, helping to focus on consistently high- or low-scoring publishers. The algorithmic rating represents a ranking score that consists of the quality of the article and the relevance of the topic.

Ordering Fields

These ordering fields allow users to control how news sources are sorted in the response. Sources can be ordered alphabetically by name, by platform, by average user rating, by average algorithmic rating, or by total article count. By default, results are returned in ascending order, but the direction can be reversed by prefixing a field with a minus symbol (-), for example -average_rating or -article_count. The default ordering is the name field, which orders the query alphabetically.

Endpoint News Categories

The third endpoint of our public API provides access to News Categories, which are referenced by the category_primary_name and category_secondary_name fields in the News Article model.

This endpoint is intended to help users build dynamic category filters, such as dropdown menus or similar UI components, without hardcoding the available categories.

GET /api/public/news-categories/

News Source Fields

The Category data models consists of only two fields: id and name. Whereas the id represents a unique identifier of a category used for filtering and ordering. The name field is the human-readable name of the category used for filtering and displaying. 

{
  "id": 1,
  "name": "Worldwide",
}

Ordering Fields

The Category endpoint supports ordering by either the category id or the category name. This allows users to retrieve categories in their default numerical order or sort them alphabetically by name. As with other endpoints, the ordering direction can be reversed by prefixing the field with a minus symbol (-). The default ordering is the id.

Pagination

Our public API allows users to define and overwrite a couple of parameters to paginate the data. In general, there are two pagination parameters that can be overwritten: page and page_size. Signum defined a standard page size of 30 tokens (1 token per news article) and a maximum number of 100 tokens. 

page
page_size
